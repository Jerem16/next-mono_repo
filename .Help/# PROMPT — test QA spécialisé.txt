# 🔎 Prompt — Setup & Organisation complète de l’environnement de tests (Next.js + Amplify)

Agis comme un expert **DevOps** et **QA** spécialisé dans les environnements **Next.js** modernes.  
Ta mission : mettre en place un système de tests **complet, modulaire et maintenable** pour un projet utilisant les technologies suivantes :

- **Framework** : Next.js v15.0.3  
- **Markdown** : react-markdown v10.1.0 + remark-gfm  
- **Styles** : Sass v1.60.0  
- **Backend** : AWS Amplify (aws-amplify v6.9.0 et @aws-amplify/ui-react)  
- **Tests unitaires & API** : Vitest  
- **Tests end-to-end (E2E/UI)** : Playwright  

---

## 🎯 Objectif

Créer **et vérifier en priorité l’environnement de tests** avant toute chose.  
Ensuite, générer un système qui :  

1. **Scaffold & organise** la structure de tests (unitaires, API, UI, e2e).  
2. **Configure tous les outils nécessaires** (Vitest, Playwright).  
3. **Détecte les tests existants** et les déplace/organise automatiquement.  
4. Permet d’exécuter facilement **tous les tests avec une seule commande** (`npm test` ou équivalent).  
5. Fonctionne avec **Next.js 15+** et prend en compte l’environnement **AWS Amplify** (mocking si besoin).  

---

## 📚 Contexte

- Le projet contient déjà des tests, mais **l’organisation est floue**.  
- On veut une **réorganisation fiable et automatisée** :  
  - `tests/unit` → tests unitaires.  
  - `tests/api` → tests backend/API.  
  - `tests/e2e` → tests Playwright.  
- Générer des **exemples de tests** si certains types manquent.  
- Fournir un **README** clair pour l’équipe (installation, exécution, ajout de nouveaux tests).
